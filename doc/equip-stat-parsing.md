# 装备属性解析

## 二进制结构

`ItemData_C.bin` 每条记录的 fields_offset 区域，每 7 字节一组 stat pair：

```
[2B code] [1B padding] [1B value_lo] [1B value_hi] [2B ?]
```

有效偏移位：`+16`, `+23`, `+30`, `+37`（最多 4 组属性）。

## 值计算公式

```python
code = data[fo + pair_off : fo + pair_off + 2]   # uint16
v1   = data[fo + pair_off + 3]                     # 低位值
v2   = data[fo + pair_off + 4]                     # 高位值

if v2 in (1, 2, 3):
    val = (v1 + v2 * 256) - 100   # 扩展值编码
elif v1 >= 100:
    val = v1 - 100                 # 普通编码
else:
    val = 0                        # 无有效值
```

原始版本只处理 `p20==2`，2026-07-05 修复为包含 `1, 2, 3`。

## 具体案例

- 流星雨手环: `+16: code=211(DEF) val=30` `+23: code=220(物减) val=5%` → `DEF+30 物减+5%`
- 铅笔剑: `+16: code=215(MATK) val=40` `+23: code=207(MaxHP) val=200` → `MATK+40 MaxHP+200`
- 寒翠絮语: `+16: code=245(抗封) val=10%` `+23: code=207(MaxHP) val=500` → `抗封+10% MaxHP+500`
