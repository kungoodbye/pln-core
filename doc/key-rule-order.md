# 钥匙规则顺序 Bug

## 问题

`getAlchemyResultCandidates()` 和 `getAlchemyResultCandidatesMulti()` 中 Step 2（钥匙规则）和 Step 3（子材覆盖）执行顺序错误。

**旧逻辑（错误）：**
```
Step 1: 材质 + 等级过滤 → 68 个候选
Step 2: 钥匙规则 — 发现副材"木材"匹配冲击弓(未)的子材 → 锁死全部候选为"含木材"
       → 晚宴礼服 (子材空) 被排除
Step 3: 子材覆盖 — 冲击弓(未) 需要 [魔性物质,木材,草类纤维]
       输入只有 [魔性物质,木材] → 冲击弓(未) 也被排除
       → 结果: 0 候选 ❌
```

**新逻辑（正确）：**
```
Step 1: 材质 + 等级过滤 → 68 个候选
Step 2: 子材覆盖过滤 — 先排除子材不全的（冲击弓(未)缺草类纤维→排除）
       → 幸存: 晚宴礼服 + 其他纯单属性装备
Step 3: 钥匙规则 — 幸存者中没有含"木材"子材的 → 不触发锁死
       → 全部幸存者通过
       → 结果: 晚宴礼服 ✅
```

## 修复合成规则

1. 先过滤掉子材不被输入材料完全覆盖的候选
2. 在幸存者中检查是否有匹配副材的子材
   - 有 → 锁定到完全匹配的候选
   - 无 → 回退到纯单属性兜底

## 影响

- `getAlchemyResultCandidates()` (2-slot 版本)
- `getAlchemyResultCandidatesMulti()` (multi-slot 版本)

两处改动完全相同：Step 2 和 Step 3 顺序互换。

## 验证

```
魔玉石(Lv21) + 垂直尾翼(Lv21,木材) + 炼金百科2

修复前: 0 候选 → 晚宴礼服不可达 ❌
修复后: [晚宴礼服(Lv27)] → 正确 ✅
```
